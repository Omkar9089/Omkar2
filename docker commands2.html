1. docker login - login to the docker hub repository

2. docker run – Runs a command in a new container.

3. docker stop – Stops one or more running containers

4. docker kill - This command kills the container by stopping its execution immediately. The difference between ‘docker kill’ and ‘docker stop’ is that ‘docker stop’ gives the container time to shutdown gracefully, in situations when it is taking too much time for getting the container to stop, one can opt to kill it

5. docker build – Builds an image form a Docker file

6. docker pull – Pulls an image or a repository from a registry

7. docker push – Pushes an image or a repository to a registry

8. docker exec – Runs a command in a run-time container

9. docker rmi - This command is used to delete an image from local storage

10. docker commit – Creates a new image from a container’s changes

11. docker ps - list the running containers

12. docker ps -a - This command is used to show all the running and exited containers

docker –version - to get the currently installed version of docker
